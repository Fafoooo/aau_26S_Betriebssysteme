Linux-Betriebssystem Fragen und Antworten
==========================================

FRAGE 1: Welche verschiedenen Dateitypen gibt es in Linux?
----------------------------------------------------------

In Linux gibt es insgesamt 7 verschiedene Dateitypen, die durch das erste Zeichen 
in der ls -l Ausgabe erkennbar sind:

1. Reguläre Dateien (-)
   - Normale Dateien wie Textdateien, Binärdateien, etc.
   - Beispiel: -rw-r--r-- datei.txt

2. Verzeichnisse (d)
   - Ordner die andere Dateien und Verzeichnisse enthalten
   - Beispiel: drwxr-xr-x verzeichnis/

3. Symbolische Links (l)
   - Verweise auf andere Dateien oder Verzeichnisse
   - Beispiel: lrwxrwxrwx link -> ziel

4. Zeichen-orientierte Gerätedateien (c)
   - Character devices wie Terminals, serielle Schnittstellen
   - Beispiel: crw-rw-rw- /dev/tty0

5. Block-orientierte Gerätedateien (b)
   - Block devices wie Festplatten, USB-Sticks
   - Beispiel: brw-rw---- /dev/sda1

6. Named Pipes/FIFOs (p)
   - Benannte Pipes für Interprozesskommunikation
   - Beispiel: prw-r--r-- meine_pipe

7. Unix Domain Sockets (s)
   - Lokale Sockets für Interprozesskommunikation
   - Beispiel: srwxrwxrwx socket_datei

FRAGE 2: ls-Befehl der nur den Monat anzeigt
-------------------------------------------

Lösung: ls -la --time-style="+%b" | awk '{print $6, $7}'

Erklärung der Befehlsteile:
- ls -la: Zeigt alle Dateien mit detaillierten Informationen
- --time-style="+%b": Formatiert das Datum so dass nur der Monatsname ausgegeben wird
- | awk '{print $6, $7}': Pipe an awk um nur die 6. und 7. Spalte auszugeben 
  (Monatsname und Dateiname)

Beispielausgabe:
Jan datei1.txt
Feb datei2.txt
Mär verzeichnis1/

FRAGE 3: Erzeugung einer 1GB großen Datei mit Zufallsdaten
----------------------------------------------------------

Befehl: dd if=/dev/urandom of=random.bin bs=1M count=1024

Erklärung der Parameter:
- dd: Data duplicator - Kommando zum kopieren und konvertieren von Daten
- if=/dev/urandom: Input file - Quelle für Zufallsdaten
- of=random.bin: Output file - Zieldatei namens "random.bin"
- bs=1M: Block size - 1 Megabyte pro Schreibvorgang
- count=1024: Anzahl der Blöcke - 1024 × 1MB = 1024MB = 1GB

Alternative mit /dev/random (langsamer aber kryptographisch sicherer):
dd if=/dev/random of=random.bin bs=1M count=1024

Der Befehl erstellt eine 1GB große Datei mit zufälligen Bytes.
